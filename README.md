# Inbox Sentinel

Inbox Sentinel is a Chrome (Manifest V3) extension designed to monitor and rescue important emails that might be hidden or lost in Gmail's "Promotions", "Social", or "Updates" categories. It works as a background worker polling your Gmail account and uses Supabase (PostgreSQL) for persistence and state synchronization.

## 🌟 Features

- **OAuth 2.0 Integration**: Secure Google OAuth sign-in and token storage in `chrome.storage.local`.
- **Gmail API Syncing**: Reads and monitors emails from Promotions, Social, and Updates categories.
- **Supabase Persistence**: Saves email metadata (sender, subject, snippet, category, timestamp) securely.
- **Background Syncing**: Uses `chrome.alarms` to poll and sync state periodically without manual intervention.
- **Browser Notifications**: Sends real-time notifications for newly discovered and synced emails.
- **Interactive Popup Dashboard**: Extension popup UI built with HTML/JS/CSS to view, refresh, configure Supabase, and mark emails as important.

## 🛠 Tech Stack

- **Extension Framework**: Chrome Extension Manifest V3 (MV3)
- **Authentication**: Google OAuth 2.0 (Identity API)
- **Database**: Supabase (PostgreSQL REST API)
- **Testing**: Jest
- **Build System**: Node.js custom scripts (`archiver`, `dotenv`)

## 🚀 Getting Started (Local Development)

### 1. Prerequisites
- **Node.js**: (v16+ recommended)
- **Google Cloud Console Account**: To create OAuth 2.0 Client IDs.
- **Supabase Account**: To host the PostgreSQL database.

### 2. Google OAuth Setup
1. Go to the [Google Cloud Console](https://console.cloud.google.com/).
2. Create a new project and navigate to **APIs & Services > Credentials**.
3. Create an **OAuth client ID** of type **Chrome App** (or **Web Application** depending on your test setup).
   - Use the extension ID generated by Chrome to define the allowed origins.
4. Note your Client ID.

### 3. Supabase Setup
1. Create a new project on [Supabase](https://supabase.com).
2. Go to the SQL Editor and execute the schema found in `db/schema.sql` to create `users`, `emails`, `notifications`, and `sync_state` tables.
3. Enable Row Level Security (RLS) policies as specified in `SUPABASE_SETUP.md` to ensure secure data access.
4. Get your Supabase Project URL and Anon Public Key from **Settings > API**.

### 4. Extension Configuration
There are two ways to configure your credentials:

**Method A: Environment Variables (Recommended for Build script)**
Create a `.env` file in the project root:
```env
GOOGLE_OAUTH_CLIENT_ID=your-google-client-id
SUPABASE_URL=your-supabase-url
SUPABASE_ANON_KEY=your-supabase-anon-key
```

**Method B: Runtime Configuration**
You can directly edit the `extension/manifest.json` `oauth2.client_id` field, and fill in `SUPABASE_URL` and `SUPABASE_ANON_KEY` in `extension/runtime-config.json` (copy from `extension/runtime-config.example.json`).

*Alternatively, the popup UI allows you to manually input the Supabase URL and Anon Key during runtime!*

### 5. Install Dependencies & Build
Install the development packages:
```bash
npm install
```

Run the build script to inject configurations and bundle the extension:
```bash
npm run build
```
This will output the final extension bundle to the `dist/extension/` directory.

### 6. Load the Extension into Chrome
1. Open Chrome and navigate to `chrome://extensions/`.
2. Enable **Developer mode** (toggle in the top right).
3. Click **Load unpacked** and select the `dist/extension/` directory.

## 🧪 Testing

The repository uses Jest for unit tests (primarily testing Supabase REST abstractions).

To run the test suite:
```bash
npm test
```

## 📦 Packaging for Production

To create a distributable `.zip` file suitable for uploading to the Chrome Web Store:
```bash
npm run package
```
This generates an `inbox-sentinel.zip` archive in the `dist/` directory.

## 📝 Important Notes

- **Security**: Never store the Supabase `service_role` key inside the extension code or runtime variables. Only use the `anon` key combined with PostgreSQL Row Level Security (RLS).
- **Privacy**: Currently, the extension only parses and persists email metadata (Message ID, Subject, Sender, Snippet, Category). It does not store the full email body or attachments to maintain a slim and secure profile.
